# MD to HTML

将 Markdown 文档转化为有格调、可本地直接访问的单文件 HTML 页面。LLM 分析文档内容和气质，推理出匹配的视觉风格候选，由用户选择后生成，所有样式和脚本内联，零外部依赖。

> ⚠️ **核心原则**：风格服务于内容。HTML 不是 Markdown 的套壳，而是对同一内容表达力的升级——视觉层次、动效节奏、排版密度，都应该与文档本身的气质匹配。所有实现优先使用原生 Web API，禁止引用 CDN 资源。生成的页面必须**移动端先行**——在手机上的阅读体验与桌面端同等重要，不是事后适配，而是默认基准。

---

## Step 1：读取并分析 Markdown 文件

用 `read_file` 工具读取用户指定的 Markdown 文件全部内容。

分析以下四个维度（分析过程不输出给用户，在内部完成）：

**① 内容类型识别**
- 技术架构 / 系统设计（大量层级结构、代码块、对比表格）
- 操作手册 / 工具指南（步骤清单、命令、参数说明）
- 概念解析 / 思想文章（连续段落、类比、观点推演）
- 数据报告 / 分析文档（大量表格、数字、指标对比）
- 叙事性文章 / 随笔（流动性散文、故事、个人经历）

**② 情绪基调识别**
- 严肃理性（技术深度、精密推导）
- 活泼轻快（工具感、效率驱动）
- 深沉哲学（思考密度高、开放性结论）
- 实用工程（直接、无废话、以结果为导向）

**③ 信息密度识别**
- 高密度：大量列表、表格、代码块、短段落
- 中密度：列表与段落混合
- 低密度：以长段落叙述为主，结构扁平

**④ 关键结构元素盘点**
- 是否有代码块（需要等宽字体 + 语法高亮区域）
- 是否有表格（需要规范的表格样式）
- 是否有多级标题（需要清晰的视觉层级）
- 是否有引用块（需要特殊的引用样式）
- 是否有图片引用（需要处理图片显示）

分析完成后，自动进入 Step 2。

---

## Step 2：推理并展示风格候选

根据 Step 1 的分析结果，推理出 **2-3 个** 匹配该文档气质的视觉风格，以以下格式展示给用户：

```
我分析了这篇文档，内容类型是 [类型]，基调偏 [基调]，信息密度 [高/中/低]。

为它推荐以下 [2/3] 种视觉风格：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
风格 A：[风格名称]
  适合理由：[一句话说明为什么这个风格适合该文档]
  视觉关键词：[3-5 个词，如：深色背景 · 等宽点缀 · 左侧锚点导航 · 代码块高亮]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
风格 B：[风格名称]
  适合理由：[一句话说明]
  视觉关键词：[3-5 个词]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（风格 C 如有）[风格名称]
  适合理由：[一句话说明]
  视觉关键词：[3-5 个词]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

请选择风格（A / B / C），或告诉我你想要的其他风格方向。
```

**等待用户回复：**
- 用户选择 A / B / C → 记录选中风格，进入 Step 3
- 用户说「不对/我想要别的风格」→ 追问：「你想要什么感觉？可以用形容词或举例说明（如：更简洁/更有科技感/像某个网站的风格）」，重新推理并展示新的候选，再次等待
- 用户说「随便/你选」→ 选择第一个推荐风格（风格 A），告知用户：「好的，我用风格 A [名称] 来做」，进入 Step 3
- 用户说「直接生成/不用选/跳过/别问了」→ 使用风格 A（第一个推荐），主题默认「跟随系统」，告知用户：「好的，用风格 A [名称] + 跟随系统主题直接生成。」，**跳过 Step 3**，直接进入 Step 4

---

## Step 3：确认明暗主题

在用户选定风格后，询问主题偏好：

```
主题偏好：🌑 暗色 / ☀️ 亮色 / 🌗 跟随系统（默认亮色，可通过页面按钮切换）？
```

**等待用户回复：**
- 用户选择暗色 / 亮色 / 跟随系统 → 记录选择，进入 Step 4
- 用户说「随便/无所谓」→ 默认选择「跟随系统」，告知用户，进入 Step 4
- 用户说「两个都要/都要能切换」→ 等同于「跟随系统」，内置亮暗切换按钮，进入 Step 4

---

## Step 4：生成 HTML 文件

根据 Markdown 内容、选定风格、选定主题，生成完整的单文件 HTML。

### 4.1 HTML 生成规范

**结构要求：**
- 所有 CSS 写在 `<style>` 标签内，内联在 HTML 文件中
- 所有 JS 写在 `<script>` 标签内，内联在 HTML 文件中
- 禁止任何 `<link rel="stylesheet" href="...">` 外部样式引用
- 禁止任何 `<script src="...">` 外部脚本引用（含 CDN）
- 禁止引用 Google Fonts、字体文件等外部字体资源，使用系统字体栈
- `<head>` 中必须包含以下移动端必要 meta 标签：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
<meta name="theme-color" content="[与页面背景色一致的色值]">
```

**系统字体栈参考：**
```css
/* 中英混排正文 */
font-family: 'SF Pro Display', -apple-system, BlinkMacSystemFont,
             'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei',
             'Helvetica Neue', Arial, sans-serif;

/* 等宽/代码 */
font-family: 'SF Mono', 'Fira Code', 'JetBrains Mono',
             'Cascadia Code', Consolas, monospace;
```

**Markdown → HTML 元素转换规范：**

| Markdown 元素 | HTML 处理方式 |
|--------------|-------------|
| `# H1` | `<h1>`，页面主标题，字号最大，通常只有一个 |
| `## H2` | `<h2>`，章节标题，带视觉分隔 |
| `### H3` | `<h3>`，小节标题 |
| 代码块 ` ``` ` | `<pre><code>`，等宽字体，带背景色区分，不依赖外部高亮库 |
| 行内代码 `` ` `` | `<code>`，背景色 + 等宽字体 |
| 表格 | `<table>`，带交替行色，hover 高亮 |
| 引用 `>` | `<blockquote>`，左边框 + 背景色区分 |
| 加粗 `**` | `<strong>` |
| 斜体 `*` | `<em>` |
| 链接 `[text](url)` | `<a>`，外部链接在新标签页打开 |
| 分割线 `---` | `<hr>`，样式化分隔线 |
| 无序列表 | `<ul><li>`，自定义列表符号样式 |
| 有序列表 | `<ol><li>`，数字样式清晰 |

**必须实现的功能：**
1. **阅读进度条**：页面顶部固定，随页面滚动实时更新进度百分比，使用 `linear-gradient` 双色渐变，`transition: width 0.1s linear` 保证流畅感，z-index 高于所有内容层
2. **平滑滚动**：`scroll-behavior: smooth`
3. **主题切换按钮**（若用户选择「跟随系统」或「两个都要」）：右下角浮动按钮（在固定控件组内），点击切换亮暗主题，使用 `localStorage` 记忆用户偏好，初始化时读取已保存偏好
4. **回到顶部按钮**：右下角浮动按钮（在固定控件组内），滚动超过 400px 后显示（`opacity: 1; pointer-events: auto`），点击平滑滚动回顶，默认隐藏（`opacity: 0; pointer-events: none`）
5. **代码块复制按钮**（若文档含代码块）：每个代码块右上角出现复制按钮，点击复制代码内容，使用 `navigator.clipboard.writeText()` 原生 API
6. **响应式布局（移动端先行）**：以下为强制要求，不是可选项：
   - 内容区最大宽度 760px，水平居中
   - 手机端（`max-width: 640px`）正文字号不小于 16px，行高不小于 1.75
   - 手机端内容区左右 padding 不小于 18px，避免文字贴边
   - 所有按钮和可交互元素的触摸目标尺寸不小于 44×44px（符合 Apple HIG 最小触摸目标规范）
   - 表格在手机端允许横向滚动（`overflow-x: auto` 包裹 `<table>`），禁止表格内容溢出屏幕
   - 代码块在手机端允许横向滚动（`overflow-x: auto`），禁止代码内容挤压变形
   - 图片默认 `max-width: 100%`，禁止图片溢出容器
   - 固定定位元素（进度条、回顶按钮、主题切换按钮）在手机端不遮挡主体内容
   - 使用 `env(safe-area-inset-*)` 处理 iPhone 刘海屏和底部安全区域
7. **章节目录（TOC Menu）**：文档大纲导航，帮助用户快速感知文档结构并跳转章节。以下为强制实现规范：
   - **自动扫描**：页面加载时扫描所有 `h2` 和 `h3` 元素，自动提取标题文字生成目录条目；`h1` 不进入目录（通常为文档主标题）
   - **锚点注入**：为每个 `h2` / `h3` 自动生成唯一 `id`（若已有则复用），用于锚点跳转
   - **桌面端 Sidebar**（`min-width: 1100px`）：固定在页面右侧，宽度 200-220px，随页面滚动高亮当前所在章节（使用 Intersection Observer），`h3` 相对 `h2` 有左缩进；Sidebar 本身支持内容滚动（`overflow-y: auto`）；若章节数量少于 3 个，不显示 Sidebar
   - **移动端底部抽屉**（`max-width: 1099px`）：在右下角浮动控件组中新增「目录」按钮（☰），点击展开从底部滑出的半屏抽屉（`position: fixed; bottom: 0; left: 0; right: 0; max-height: 65vh`），抽屉顶部有拖拽条和「× 关闭」按钮，条目点击后自动关闭抽屉并跳转；抽屉打开时，背景遮罩（`backdrop-filter: blur`）阻止底层滚动
   - **当前位置高亮**：目录条目通过 Intersection Observer 感知当前阅读位置，高亮对应条目（accent 色文字 + 左侧竖线）
   - **平滑跳转**：点击目录条目使用 `element.scrollIntoView({ behavior: 'smooth', block: 'start' })` 跳转，跳转后有轻微偏移补偿（`scroll-margin-top: 24px`）以避免被固定进度条遮挡
   - **零依赖**：完全用原生 JS 实现，禁止引入任何第三方滚动库或目录库

**禁止的实现方式：**
- ❌ 禁止引入 highlight.js、Prism.js 等外部代码高亮库（CDN 形式）
- ❌ 禁止引入 animate.css、AOS 等外部动画库
- ❌ 禁止引入任何 UI 框架（Bootstrap、Tailwind CDN 版等）
- ❌ 可以将 highlight.js 等库的核心逻辑以内联形式写入，但仅在文档有大量代码且必要时使用
- ❌ **禁止用 `h2::before` / `h3::before` 伪元素给标题加 `##` / `###` 文字装饰**——这会把 Markdown 原始语法「漏」到渲染后的页面上，产生视觉噪音。标题就是标题，层级通过字号、字重、下边框体现，不需要符号前缀。若需要「终端感」装饰，应通过独立的 badge 元素（如 `<span class="step-num">STEP 01</span>`）实现，而不是 CSS 伪元素
- ❌ **禁止在自定义列表符号（`::before` 伪元素）的同时保留浏览器默认 `list-style`**——两者会同时渲染导致符号 double。凡是用 `::before` 自定义列表符号，必须同时在 `ul, ol` 和 `li` 上显式设置 `list-style: none`

**允许但需谨慎：**
- ✅ 使用 CSS Custom Properties（变量）管理主题色
- ✅ 使用 CSS `@media (prefers-color-scheme)` 响应系统主题
- ✅ 使用 Intersection Observer API 实现滚动动画
- ✅ 使用 `navigator.clipboard` API 实现复制功能
- ✅ 使用 `@media (hover: none)` 区分触摸设备和鼠标设备，触摸设备上禁用 hover 动效（避免 iOS 点击残留高亮）
- ✅ 使用 `-webkit-tap-highlight-color: transparent` 消除 iOS 点击蓝色闪烁
- ✅ 使用 `touch-action: manipulation` 消除双击缩放延迟

### 4.2 风格参考库（LLM 根据分析结果选择）

以下是各内容类型推荐的风格方向，LLM 可在此基础上延伸：

**技术架构 / 系统设计文档 → 推荐方向：**
- 极简科技感：深色 `#0a0e1a` 背景，蓝色 `#63b3ed` 强调色，等宽字体点缀，左侧锚点导航
- 工程白板：白色背景，浅灰边框，Sans-serif，紧凑间距，代码块深灰底
- 信息图谱：深色背景，多色强调（每个层级不同色），节点感布局

**操作手册 / 工具指南 → 推荐方向：**
- 清单卡片：白色/浅灰底，步骤编号大字突出，代码块和普通文本对比鲜明
- 终端风格：全黑背景，等宽字体为主，绿色 `#68d391` 强调，极客感

**概念解析 / 思想文章 → 推荐方向：**
- 学术白皮书：米白 `#fafaf8` 背景，衬线字体正文，深色标题，适合长阅读
- 杂志排版：大字号标题，宽行距，引用块突出，段落间距宽松

**数据报告 / 分析文档 → 推荐方向：**
- 仪表盘风格：深色背景，表格突出，数字强调，多色分类
- 商务简报：白色背景，蓝色强调，表格样式规范，适合打印

**叙事性文章 / 随笔 → 推荐方向：**
- 手记本：米黄背景，衬线字体，软阴影，温暖色调
- 现代博客：白色背景，大行距，无衬线，图片全宽展示

### 4.3 输出文件

**文件命名规则：**
- 取输入 Markdown 文件名（去掉 `.md` 后缀），加 `.html` 后缀
- 输入 `my-doc.md` → 输出 `my-doc.html`
- 输入 `20260512-arch.md` → 输出 `20260512-arch.html`

**输出路径：**
- 与输入 Markdown 文件同目录

**禁止的输出方式：**
- ❌ 禁止只在对话中展示 HTML 代码片段而不写入文件
- ❌ 禁止生成依赖外部 CDN 资源的 HTML（违反零依赖原则）
- ❌ 禁止输出到与输入文件不同的目录（除非用户明确指定其他路径）

**执行步骤：**
1. 在内部将 Markdown 内容转换为对应 HTML 元素（参考 4.1 转换规范）
2. 将完整 HTML 内容（含内联 CSS 和 JS）用 `write` 工具写入目标路径
3. 输出以下内容告知用户，**流程结束**：

```
✅ 已生成：[输出文件路径]
风格：[选定风格名称] · 主题：[暗色/亮色/跟随系统] · 文件大小：约 [估算大小]

用浏览器直接打开即可：
  open [输出文件路径]

如需调整风格或重新生成，告知我即可。
```

> 流程在输出上述告知后正式结束。如用户继续回复「调整风格」「重新生成」等，重新从 Step 2 开始执行（无需重新读取文件，复用已分析的内容）。

## 边界情况

### Markdown 文件过长（超过 500 行）

生成前提示用户：

「这篇文档较长（约 [行数] 行），生成的 HTML 文件体积可能较大（>200KB）。继续生成，还是你想先截取某个章节？」

- 用户说「继续」→ 正常生成
- 用户说「截取章节」→ 让用户指定起止标题，只处理该范围的内容，文件名加章节后缀（如 `my-doc-chapter2.html`）

### 纯代码文档（几乎全是代码块）

如果文档中代码块比例超过 70%，风格推理时必须将「终端风格」或「工程白板」作为候选之一，并说明理由：

「这篇文档以代码为主，推荐代码可读性更强的风格。」

### 无标题文档（缺少 `#` 标题）

如果文档没有任何 `#` 标题，用文件名作为页面 `<title>` 和视觉主标题，并提示：

「文档没有检测到标题，我用文件名 [filename] 作为页面标题。如需修改，告知我。」

### 含外部图片引用

如果 Markdown 含 `![](http://...)` 形式的远程图片，提示：

「文档包含 [N] 处远程图片引用，在断网环境下可能无法显示。如需离线可用，建议将图片下载到本地后替换路径。我会在生成的 HTML 中保留原始引用，并在图片处加注 `<!-- 远程图片，离线不可用 -->`。」

→ 不阻塞生成，直接继续。

### 含本地图片引用

如果 Markdown 含 `![](./images/xxx.png)` 形式的本地图片，生成后告知：

「HTML 文件引用了 [N] 处本地图片，请确保图片文件与 HTML 文件保持相对路径关系，否则图片将无法显示。」

### Markdown 解析遇到不支持的语法

如果文档包含 LaTex 数学公式（`$...$` 或 `$$...$$`）、Mermaid 图表（` ```mermaid `）等特殊语法，生成前告知：

「检测到文档包含 [数学公式/Mermaid 图表]，这类内容需要第三方库渲染，在零依赖模式下会以代码块形式原样展示。你希望：
A. 保持零依赖，以代码块展示原始语法
B. 内联 [MathJax 核心脚本 / Mermaid 核心脚本]（约 [大小]，文件会更大但功能完整）」

- 用户选 A → 正常生成，特殊语法以 `<pre><code>` 展示
- 用户选 B → 将对应库的核心代码内联进 HTML（注意：内联的库代码必须是已知可离线运行的版本，不能是远程 CDN 引用）
