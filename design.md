# PRD：SVG 画布工具

## 1. 项目概述

| 字段 | 内容 |
|------|------|
| 项目名称 | SVG 画布（svg-canvas） |
| 所属项目 | webpages 工具集 |
| 定位 | 纯静态 · 零后端 · 浏览器本地处理的轻量级 SVG 矢量绘图工具 |
| 目标用户 | 通用用户（开发者、设计师、普通用户均适用） |
| 文档状态 | 初稿 |

---

## 2. 产品目标

为用户提供一款无需安装、开箱即用的浏览器端 SVG 矢量图绘制工具。用户可以通过多种自由路径绘制模式构建矢量图形，配置样式属性，并将结果导出为多种格式使用。

**核心价值**：
- 零门槛：无需注册、无需安装，打开即用
- 自由绘制：支持多种路径绘制模式（折线、贝塞尔曲线、自由手绘）
- 可编辑：绘制完成后支持节点拖拽、控制点调整等完整编辑能力
- 多格式导出：SVG 文件、PNG 图片、SVG 代码、节点数据 JSON

## 3. 功能范围

### 3.1 核心功能（必须实现）

#### 3.1.1 绘图模式

| 模式 | 描述 |
|------|------|
| **折线模式** | 依次点击节点，连成折线或多边形；双击结束绘制；点击起点可闭合路径 |
| **贝塞尔曲线模式** | 点击添加锚点，拖拽生成控制柄，绘制平滑曲线 |
| **自由手绘模式** | 鼠标/触控板按住拖动，轨迹自动转为简化的 SVG `<path>` |

工具栏提供三种模式切换，同一时刻只能处于一种绘制模式。

#### 3.1.2 编辑能力

- **选择工具**：点击选中已有路径；框选多个路径
- **节点编辑**：选中路径后显示所有节点，支持拖拽移动单个节点
- **控制点调整**：贝塞尔曲线模式绘制的路径，选中后可拖拽控制柄调整弧度
- **删除**：选中后按 `Delete` / `Backspace` 删除
- **撤销 / 重做**：`Ctrl+Z` / `Ctrl+Shift+Z`（Mac：`Cmd+Z` / `Cmd+Shift+Z`），支持最多 50 步
- **层级调整**：右键菜单或工具栏按钮，支持「前移一层」和「后移一层」

#### 3.1.3 样式配置

以下属性在右侧属性面板中配置，针对**当前选中的路径**生效（未选中时为全局默认值）：

| 属性 | 说明 |
|------|------|
| 线条颜色 | 颜色选择器，支持 HEX / RGB 输入 |
| 线条宽度 | 数字输入框，范围 0.5 ~ 20px |
| 虚线样式 | 下拉选择：实线 / 短虚线 / 长虚线 / 点线 |
| 填充色 | 颜色选择器（仅对闭合路径生效），支持「无填充」选项 |
| 透明度 | 滑块，0% ~ 100% |

**画布属性**（始终可配置）：

| 属性 | 说明 |
|------|------|
| 背景色 | 颜色选择器，支持透明（checkerboard 背景表示透明） |
| 画布尺寸 | 宽 × 高，预设常用尺寸（800×600、1920×1080 等），也可自定义输入 |

#### 3.1.4 导出 / 下载

| 格式 | 行为 |
|------|------|
| SVG 文件 | 下载 `.svg` 文件，保留矢量信息 |
| PNG 图片 | 基于 Canvas 渲染后下载 `.png`，支持选择分辨率倍数（1x / 2x） |
| SVG 代码 | 将 SVG 源码复制到剪贴板，弹出 Toast 提示 |
| 节点数据 JSON | 导出包含所有路径节点坐标和样式信息的 JSON，用于后续重新导入 |

#### 3.1.5 本地持久化

- 用户绘制内容**自动保存**到 `localStorage`（防抖 500ms，防止频繁写入）
- 页面刷新后自动恢复上次绘制状态
- 支持**手动导入 JSON**（拖拽文件或点击「导入」按钮）还原历史画布
- 提供「清空画布」按钮，二次确认后清空内容并清除 localStorage

---

### 3.2 非功能需求

- **纯静态**：无需服务端，所有逻辑在浏览器本地完成
- **性能**：路径数量在 500 条以内时，绘制和编辑操作帧率不低于 30fps
- **兼容性**：支持 Chrome 90+、Firefox 88+、Safari 14+（不需要兼容 IE）
- **响应式**：画布区域自适应窗口大小，工具栏在小屏下可折叠

---

### 3.3 不在范围内（Out of Scope）

- ❌ 标准形状（矩形、圆形、箭头等）—— 专注自由路径绘制
- ❌ 文字标注
- ❌ 图层面板（不需要命名图层，只需前移/后移）
- ❌ 云端同步 / 账号系统
- ❌ 协同编辑

## 4. 用户交互流程

### 4.1 主流程：绘制并导出

```
进入页面
  └─> 自动恢复 localStorage 中的历史内容（若有）
      └─> 选择绘图模式（折线 / 贝塞尔 / 手绘）
          └─> 在画布上绘制路径
              └─> [可选] 切换选择工具 → 选中路径 → 调整样式 / 节点
                  └─> 点击导出按钮 → 选择格式 → 完成
```

### 4.2 导入流程

```
点击「导入 JSON」或拖拽 .json 文件到画布
  └─> 解析节点数据
      └─> [成功] 渲染到画布，覆盖或追加（弹窗询问用户）
      └─> [失败] Toast 提示解析错误
```

## 7. 技术实现约束

- **纯静态**：HTML + CSS + 原生 JavaScript，无需打包工具和 Node.js 依赖
- **SVG 操作**：直接操作 SVG DOM（不引入 D3.js 等重型库）；如需贝塞尔曲线辅助可使用轻量工具函数
- **Canvas 导出**：使用 `canvg` 或自行实现 SVG → Canvas 渲染，导出 PNG
- **本地存储**：使用 `localStorage`，Key 为 `svg-canvas-data`
- **文件结构**：
  ```
  svg-canvas/
    index.html      # 单文件或入口
    style.css       # 样式（或内联）
    app.js          # 核心逻辑（或内联）
  ```

### 8.1 功能验收

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| F-01 | 折线模式绘制 | 点击画布依次添加节点，形成折线；双击结束；点击起点闭合 |
| F-02 | 贝塞尔模式绘制 | 点击添加锚点，拖拽产生弧度，路径为平滑曲线 |
| F-03 | 自由手绘 | 鼠标拖动生成路径，松开后路径固定 |

| F-05 | 撤销/重做 | Ctrl+Z 可逐步撤销，最多 50 步 |
| F-06 | 样式配置 | 修改颜色/宽度/虚线/填充/透明度后画面实时更新 |

| 编号 | 验收项 | 通过标准 |

## 9. 优先级与里程碑

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| P0 | 折线模式绘制 + 选择/删除 + 撤销/重做 + SVG/PNG 导出 | 必须 |
| P1 | 贝塞尔曲线模式 + 自由手绘模式 + 节点编辑 | 高 |
| P2 | 完整样式配置面板 + 背景色/画布尺寸 | 高 |
| P3 | JSON 导入导出 + localStorage 自动保存 | 中 |
| P4 | 层级调整 + 首页卡片集成 | 低 |
